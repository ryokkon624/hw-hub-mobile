# #1 ログイン モバイル仕様

## 1. 概要

メール/パスワード、またはGoogleアカウントでログインする画面。

モバイル版でのGoogleログインはWeb版のリダイレクト方式と異なり、`google_sign_in` パッケージ（ネイティブSDK）を使ってIDトークンを取得し、バックエンドのモバイル専用エンドポイントに送信する。

## 3. 機能仕様

### 3.1 メール/パスワードログイン

| 項目 | 仕様 |
|---|---|
| メールアドレス | 必須。キーボードは `emailAddress` タイプ |
| パスワード | 必須。入力は非表示（`obscureText: true`） |
| ログインボタン | メール・パスワード両方入力済みの場合のみ活性 |
| ローディング中 | ボタンをスピナー表示に切り替え、再タップ不可 |
| エラー表示 | 認証失敗時（401）はフォーム下部にインラインメッセージを表示 |

#### バリデーション（送信前）

| フィールド | ルール | エラー文 |
|---|---|---|
| メール | 必須 | 入力してください |
| パスワード | 必須 | 入力してください |

#### ログイン後の遷移

- `redirectAfterLogin` が保存されている場合はそのルートへ遷移（招待リンク経由など）
- それ以外はホーム画面へ遷移

### 3.2 Googleログイン（モバイル固有）

Web版（リダイレクト方式）とは実装が異なる。

| ステップ | 処理 |
|---|---|
| 1 | `google_sign_in` でネイティブGoogleサインインを開始 |
| 2 | Googleのネイティブ選択UIを表示（OS標準） |
| 3 | 成功時、`GoogleSignInAccount` からIDトークンを取得 |
| 4 | `POST /api/auth/google/mobile` にIDトークンを送信 |
| 5 | レスポンスのJWTをセキュアストレージに保存してログイン完了 |
| 6 | ホーム画面へ遷移（`redirectAfterLogin` があればそちら優先） |

- キャンセル・失敗時はスナックバーでエラー通知
- `email already used for local account` エラー時は「このメールはパスワード認証で登録済みです」旨をスナックバー表示

### 3.3 パスワード忘れ遷移

- 「パスワードを忘れた方」をタップするとパスワード忘れ画面へ遷移
- メールアドレス入力済みの場合は、そのメールアドレスをクエリパラメータ（`?email=xxx`）として渡す

### 3.4 ログイン済みの場合

アプリ起動時に認証済みと判定された場合、このページは表示せずにホームへリダイレクトする。

## 4. APIエンドポイント

| メソッド | エンドポイント | 用途 |
|---|---|---|
| POST | `/api/auth/login` | メール/パスワードログイン |
| POST | `/api/auth/google/mobile` | Googleログイン（モバイル専用・新規エンドポイント） |

### POST /api/auth/login

```json
// Request
{ "email": "user@example.com", "password": "password123" }

// Response 200
{ "token": "<JWT>", "user": { "userId": 1, "email": "...", "displayName": "..." } }
```

### POST /api/auth/google/mobile

```json
// Request
{ "idToken": "<Google IDトークン>" }

// Response 200
{ "token": "<JWT>", "user": { "userId": 1, "email": "...", "displayName": "..." } }
```
