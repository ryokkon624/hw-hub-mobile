# #25 問い合わせ詳細 モバイル仕様

## 1. 概要

問い合わせのメッセージスレッド表示・返信・解決/エスカレート操作を行う画面。Web版SP版と同等の機能をそのまま踏襲する。

## 3. 機能仕様

### 3.1 初期表示

- `GET /api/inquiries/{inquiryId}` で詳細を取得
- 取得失敗時は #23 問い合わせ一覧 へリダイレクト

### 3.2 メッセージ表示

| 送信者タイプ | 表示位置 | 色 |
|---|---|---|
| USER（あなた） | 右寄せ | 青（プライマリカラー） |
| AI | 左寄せ | バイオレット系 |
| STAFF | 左寄せ | グリーン系 |

- 各メッセージに送信者ラベルと送信日時を表示

### 3.3 ステータス説明文

スレッド末尾（返信フォーム直上）にステータスに応じた説明文を表示:

| ステータス | 説明文の内容 |
|---|---|
| OPEN | 受付中。AIが確認しています |
| AI_ANSWERED | AIが回答しました。解決した/しなかったを選択してください |
| PENDING_STAFF | スタッフに引き継ぎました |
| STAFF_ANSWERED | スタッフが回答しました。解決した場合はクローズしてください |
| CLOSED | このお問い合わせは解決済みです |

### 3.4 返信フォーム

- CLOSED 以外のステータスで表示
- テキストが空の場合は「送信」ボタン無効
- `POST /api/inquiries/{inquiryId}/messages` で送信
- 送信成功後スナックバーで通知し、テキストフィールドをクリア

### 3.5 アクションボタン

| ステータス | ボタン | 処理 |
|---|---|---|
| AI_ANSWERED | 解決した | 確認ダイアログ後 `POST /api/inquiries/{id}/close` |
| AI_ANSWERED | スタッフへ引き継ぐ | 確認ダイアログ後 `POST /api/inquiries/{id}/escalate` |
| STAFF_ANSWERED | 解決した | 確認ダイアログ後 `POST /api/inquiries/{id}/close` |

---

## 4. API

| メソッド | エンドポイント | 用途 |
|---|---|---|
| GET | `/api/inquiries/{inquiryId}` | 問い合わせ詳細取得 |
| POST | `/api/inquiries/{inquiryId}/messages` | 返信追加 |
| POST | `/api/inquiries/{inquiryId}/close` | 解決としてクローズ |
| POST | `/api/inquiries/{inquiryId}/escalate` | スタッフへエスカレート |
