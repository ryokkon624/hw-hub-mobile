# #14 買い物アイテム作成 モバイル仕様

## 1. 概要

新しい買い物アイテムを登録する画面。過去の購入履歴・お気に入りから選択して登録することも可能。

Web版と同等の機能をそのまま踏襲する。

## 3. 機能仕様

### 3.1 フォーム

| フィールド | 必須 | 内容 |
|---|---|---|
| 品名 | ✅ | テキスト入力 |
| メモ | | テキスト入力（複数行） |
| 購入場所 | | プルダウン選択。世帯に登録されている購入場所を表示 |
| お気に入りにする | | チェックボックス |
| 画像 | | 1枚まで。image_pickerでカメラ/フォトライブラリから選択 |

### 3.2 過去の購入から選ぶ

ボタンタップでモーダル（ボトムシート）を表示。

**モーダル内**
- 品名で検索（テキスト入力）
- 購入場所フィルタ（すべて / スーパー / ドラッグストア / オンライン）
- 期間フィルタ（全期間 / 過去30日 / 過去90日 / 過去1年）
- 購入済みアイテム一覧（品名 / 購入場所 / 購入日 / 購入回数）
- 「追加」ボタンタップで各フィールドに値をセット、モーダルを閉じる

### 3.3 お気に入りから選ぶ

ボタンタップでモーダル（ボトムシート）を表示。

**モーダル内**
- 品名で検索（テキスト入力）
- 購入場所フィルタ（すべて / スーパー / ドラッグストア / オンライン）
- お気に入りアイテム一覧（品名 / 購入場所 / メモ / 画像有無）
- 「選ぶ」ボタンタップで各フィールドに値をセット、モーダルを閉じる

### 3.4 登録処理

「追加する」ボタンタップで以下を順に実行する。

1. バリデーション（品名必須）
2. アイテム登録 `POST /api/shopping-items`
3. 画像が選択されている場合:
   - Presigned URL取得 `POST /api/shopping-items/{itemId}/attachments/upload-url`
   - S3へ直接PUT
   - メタデータ登録 `POST /api/shopping-items/{itemId}/attachments`
4. 完了後、#13買い物リスト画面に戻る

---

## 4. API

| メソッド | エンドポイント | 用途 |
|---|---|---|
| POST | `/api/households/{id}/shopping-items` | アイテム新規登録 |
| GET | `/api/households/{id}/shopping-items/history-suggestions` | 過去の購入履歴取得 |
| GET | `/api/households/{id}/shopping-items/favorites` | お気に入り一覧取得 |
| POST | `/api/shopping-items/{itemId}/attachments/upload-url` | 画像アップロード用Presigned URL発行 |
| POST | `/api/shopping-items/{itemId}/attachments` | 画像メタデータ登録 |
