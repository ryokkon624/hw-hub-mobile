# #2 サインアップ モバイル仕様

## 1. 概要

新規アカウントを作成する画面。メールアドレス・表示名・パスワード・利用言語を入力して登録する。

登録後はメール認証が必要なため、認証メール待機画面 (#3) へ遷移する。

Web版との差異はなく、レイアウトをモバイル向けに縦1カラムに調整する程度。

## 3. 機能仕様

### 3.1 フォームバリデーション

#### 送信前バリデーション（ボタン活性制御）

| フィールド | ルール | ボタン非活性条件 |
|---|---|---|
| メール | 必須・未入力なら非活性 | 空白のみ |
| 表示名 | 必須・未入力なら非活性 | 空白のみ |
| パスワード | 8文字以上・未満なら非活性 | 7文字以下 |
| パスワード確認 | パスワードと一致しなければ非活性 | 不一致 |

#### 送信時（APIエラー）

| HTTPステータス / エラーコード | 表示 |
|---|---|
| 409 Conflict（メール重複） | 「このメールアドレスはすでに登録されています」 |
| その他 4xx / 5xx | 「登録に失敗しました。もう一度お試しください」 |

### 3.2 利用言語

- 選択肢: 日本語（`ja`）、英語（`en`）、スペイン語（`es`）
- デフォルト: 端末のロケールに応じて自動選択（対応外は `ja`）

### 3.3 登録後の処理

| `emailVerificationRequired` | 処理 |
|---|---|
| `true` | 認証メール待機画面 (#3) へ遷移（メールアドレスをパラメータで渡す） |
| `false` | 即ログイン扱いとしてホーム (#10) へ遷移（dev/stg環境のみ） |

### 3.4 ローディング中

- 送信ボタンをスピナー表示に切り替え、再タップ不可

## 4. APIエンドポイント

| メソッド | エンドポイント | 用途 |
|---|---|---|
| POST | `/api/auth/register` | 新規アカウント登録 |

### POST /api/auth/register

```json
// Request
{
  "email": "user@example.com",
  "password": "password123",
  "displayName": "田中 太郎",
  "locale": "ja",
  "invitationToken": null
}

// Response 200
{
  "emailVerificationRequired": true,
  "token": null,
  "user": { "userId": 1, "email": "...", "displayName": "..." },
  "verificationExpiresAt": "2026-04-27T12:00:00Z"
}
```

> `invitationToken` は招待リンクからサインアップした場合のみセット（通常は `null`）。
