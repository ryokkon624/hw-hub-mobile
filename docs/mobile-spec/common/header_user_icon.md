# 共通ヘッダー ユーザーアイコン 仕様

## 1. 概要

全メイン画面（ボトムナビを持つ画面）のヘッダー右端に配置するユーザーアイコン。
タップでポップオーバーメニューを表示し、設定画面へのショートカットとログアウト操作を提供する。

---

## 2. 配置対象画面

ボトムナビを持つメイン画面のみに配置する（← 戻るを持つサブ画面には配置しない）。

| 画面 | # |
|---|---|
| ホーム | #10 |
| 家事分担 | #11 |
| My Tasks | #12 |
| 買い物リスト | #13 |
| 通知センター | #16 |
| 設定トップ | #17 |

ヘッダー右端の構成（左から順）: `🔔 ベルアイコン` → `👤 ユーザーアイコン`

---

## 3. アイコン表示

- 現在ログイン中のユーザーのプロフィール画像を円形で表示する
- プロフィール画像未設定の場合は `m_user.display_name` のイニシャルをテキストで表示する（例: 「田中 太郎」→「田中」、「Taro Tanaka」→「Ta」）
- 画像URLは既存の認証状態（AuthState または UserProfileState）から取得する

---

## 4. タップ時の動作

タップでポップオーバーメニューを表示する。

```
┌─────────────────┐
│  ⚙️  設定       │
├─────────────────┤
│  🚪  ログアウト  │
└─────────────────┘
```

### 4.1 「設定」タップ

→ #18 アカウント設定 に遷移する。

### 4.2 「ログアウト」タップ

1. 確認ダイアログを表示する（「ログアウトしますか？」／「はい」「キャンセル」）
2. 「はい」選択後、ローカルの認証情報（JWT・リフレッシュトークン）を削除する
3. #1 ログイン画面に遷移する（ナビゲーションスタックをクリア）

---

## 5. 状態管理

- 画像URLは既存の `AuthState` または `UserProfileState` から取得する
- ログアウト処理は既存の `AuthNotifier.logout()` を流用する
