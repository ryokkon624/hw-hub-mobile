# #11 家事分担 モバイル仕様

## 1. 概要

世帯の家事タスクの担当者を調整する画面。未割当の家事や、負荷が偏っている家事をここで割り振る。

モバイル版では以下の2モードで構成する。

| モード | 概要 |
|---|---|
| **通常リストモード** | 全タスクをリスト表示。左スワイプで自分に割り当て、右スワイプでメンバー選択モーダルを起動 |
| **スワイプモード** | タスクを1枚ずつ表示し、左スワイプで自分に割り当て、右スワイプで次へ。自分にタスクを割り当てることに特化したフロー |

## 3. 機能仕様

### 3.1 メンバー別未対応件数サマリ

- 世帯メンバー全員分の未対応タスク件数を横スクロールで表示
- 「未割当」を先頭に固定表示
- 現在ログイン中のユーザーをハイライト表示

### 3.2 フィルタ

| フィルタ | 表示対象 | 主なユースケース |
|---|---|---|
| すべて | 全タスク | 全体状況の把握、誰かに押し付けたい時 |
| 未割当のみ | 担当者が未割当のタスクのみ | 未割当タスクを誰かに割り当てる時 |
| 自分＋未割当 | 自分担当 + 未割当のタスク | 自分のタスクを誰かに押し付けつつ未割当も割り当てる時 |

### 3.3 タスクカード（通常リストモード）

各カードに以下を表示・操作できる。

| 要素 | 内容 |
|---|---|
| 担当バッジ | 現在の担当者（未割当 or メンバー名） |
| 家事名 | |
| 実施日 | |

スワイプ操作:

| 操作 | 内容 |
|---|---|
| 左スワイプ | そのタスクを即時自分に割り当て |
| 右スワイプ | メンバー選択モーダルを起動。任意のメンバーに割り当て（未割当に戻すことも可） |

### 3.4 過去の未割当を一括スキップ

- 過去日付の未割当タスクが存在する場合のみボタンを表示
- タップで確認ダイアログを表示
- 確認後、対象タスクを一括スキップ（タスクのステータスをスキップに変更）

### 3.5 スワイプモード

#### 起動

| ボタン | スワイプ対象 |
|---|---|
| [未割当を自分に割り当てる ▶] | 未割当タスクのみ |
| [他メンバーのタスクを奪う ▶] | 他メンバー担当タスクのみ |

#### 操作

| 操作 | 内容 |
|---|---|
| 左スワイプ | そのタスクを自分に割り当てて次のカードへ |
| 右スワイプ | 割り当てず次のカードへ（タスクのステータス変更なし） |
| ← （戻るボタン） | スワイプモードを終了し通常リストモードに戻る |

#### 表示内容

- 家事名
- 実施日
- 現在の担当（未割当 or メンバー名）
- 進捗（例: 3 / 40）

#### 終了条件

- 全カードを消化した場合 → 完了メッセージを表示して通常リストモードに戻る
- 戻るボタンで任意のタイミングで終了可能

## 4. API

既存APIをそのまま利用する。

| メソッド | エンドポイント | 用途 |
|---|---|---|
| GET | `/api/housework-tasks` | タスク全件取得。フィルタはフロント側で実施 |
| GET | `/api/households/{householdId}/members` | 世帯メンバー一覧取得（サマリ表示用） |
| PATCH | `/api/housework-tasks/{taskId}/assign` | 担当者変更（左スワイプ自己割り当て / メンバー選択モーダル / スワイプモード割り当て共通） |
| PATCH | `/api/housework-tasks/bulk-status` | 過去の未割当を一括スキップ |

**取得件数について**

バッチが日次で未来7〜10日分のタスクを生成する仕組み、かつ、過去分は8日以前は取得しないため全件取得でも件数は限定的。ページングは不要。

**フィルタ処理（フロント側）**

| フィルタ | 絞り込み条件 |
|---|---|
| すべて | 絞り込みなし |
| 未割当のみ | `assignee == null` |
| 自分＋未割当 | `assignee == me || assignee == null` |

## 5. 状態管理

```
HouseworkAssignState
├── tasks: List<HouseworkTask>         // タスク一覧
├── members: List<HouseholdMember>     // 世帯メンバー（サマリ用）
├── filter: AssignFilter               // すべて / 未割当 / 自分+未割当
├── mode: AssignMode                   // list / swipe
├── swipeTarget: SwipeTarget           // unassigned / others
├── swipeIndex: int                    // スワイプモードの現在位置
└── isLoading: bool
```

## 7. モバイル固有の考慮点

### スワイプ操作

**通常リストモード**（各カードのスワイプ）:

| 操作 | 背景色 | ラベル |
|---|---|---|
| 左スワイプ（自分に割り当て） | sky-500（青）| 自分にする |
| 右スワイプ（メンバー選択） | amber-500（橙）| メンバー選択 |

**スワイプモード**（Tinder風）:

| 操作 | 背景色 | ラベル |
|---|---|---|
| 左スワイプ（自分に割り当て） | sky-500（青）| 自分にする |
| 右スワイプ（次へ） | グレー（slate-400）| 次へ |

- スワイプ中はカードが追従してドラッグし、しきい値（カード幅の30%）を超えたらアクション確定
- スワイプモード終了後は通常リストを最新状態に再取得する
- タスク件数が0件の場合、スワイプモード起動ボタンはグレーアウト（非活性）
