# #16 通知センター モバイル仕様

## 1. 概要

世帯に関する通知を一覧表示する画面。通知タップで該当画面に遷移する。
ヘッダーのベルアイコンからのポップオーバーと、通知センター画面の2つのエントリーポイントを持つ。

---

## 2. エントリーポイント

### 2.1 ベルアイコン（ポップオーバー）

ヘッダーのベルアイコンタップで表示。最新20件を取得・表示。

```
┌───────────────────────────────┐
│ 通知センター        すべてみる │
├───────────────────────────────┤
│ ● 他メンバーがあなたのタスクを │  ← 未読（●）
│   奪いました                  │
│   ママがあなたのタスクを...    │
│   2026/4/14 14:01:29          │
├───────────────────────────────┤
│   買い物アイテムが追加されました│  ← 既読（●なし）
│   オリーブオイルが追加されました│
│   2026/4/13 10:22:00          │
└───────────────────────────────┘
```

- 表示と同時に全件既読に更新（`markRead=true`）
- 「すべてみる」タップで通知センター画面へ遷移
- 各通知タップで該当画面に遷移（ポップオーバーを閉じてから遷移）

### 2.2 通知センター画面（#16）

ボトムナビ等から直接遷移。最新50件を取得・表示。

## 4. 機能仕様

### 4.1 通知一覧

| 要素 | 内容 |
|---|---|
| 未読インジケーター | ●（未読）/ なし（既読） |
| タイトル | 通知のタイトル |
| 本文 | 通知の詳細テキスト |
| 日時 | 通知作成日時 |
| > アイコン | linkTypeがNONE以外の場合のみ表示 |

### 4.2 既読処理

| エントリーポイント | タイミング | limit |
|---|---|---|
| ベルアイコン（ポップオーバー） | 表示時 | 20件 |
| 通知センター画面 | 表示時 | 50件 |

どちらも取得時に`markRead=true`で全件既読に更新する。

### 4.3 通知タップ時の遷移

`linkType`に応じて以下の画面に遷移する。

| linkType | 遷移先 |
|---|---|
| NONE | 遷移なし |
| MY_TASKS | #12 My Tasks |
| HOUSEHOLD | #19 世帯設定 |
| SETTINGS | #18 アカウント設定 |
| INVITATION | #19 世帯設定 |
| INQUIRY | #25 問い合わせ詳細（`linkId`を渡す） |

### 4.4 更新ボタン

通知一覧を再取得する（`markRead=true`で取得）。

### 4.5 ベルアイコンの未読件数バッジ

ヘッダーのベルアイコンに未読件数をバッジで表示する（例: 🔔 **6**）。
未読件数が0の場合はバッジを非表示にする。

**更新タイミング**
- アプリ起動時
- フォアグラウンド復帰時
- ポップオーバー / 通知センター表示後（既読になるため0にリセット）

---

## 5. API

| メソッド | エンドポイント | パラメータ | 用途 |
|---|---|---|---|
| GET | `/api/notifications` | `limit=20&markRead=true` | ベルポップオーバー用取得 |
| GET | `/api/notifications` | `limit=50&markRead=true` | 通知センター用取得 |
| GET | `/api/notifications/unread-count` | - | 未読件数取得（ベルバッジ表示用） |

---

## 6. 状態管理

```
NotificationState
├── notifications: List<NotificationModel>  // 通知一覧
├── unreadCount: int                         // 未読件数（ベルバッジ表示用）
└── isLoading: bool
```
