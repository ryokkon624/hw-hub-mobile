# #12 My Tasks モバイル仕様

## 1. 概要

自分に割り当てられている家事タスクの一覧画面。タスクの完了・スキップをスワイプ操作で登録する。

Web版SP版と同等の機能をベースに、モバイルではスワイプ操作に特化したUI構成とする。

## 3. 機能仕様

### 3.1 過去の家事セクション

今日より前に対応する予定だった未対応タスクを日付ごとにグループ表示する。

| 要素 | 内容 |
|---|---|
| セクションヘッダー | ⚠️アイコン + 「過去の家事」ラベル + 説明文 |
| すべて完了にするボタン | タップで確認ダイアログを表示。確認後、過去の未対応タスクを全件完了に更新 |
| 日付グループ | 日付（曜日）+ 件数 |
| タスクカード | 家事名 + スワイプ操作（完了 / スキップ） |

過去の未対応タスクが0件の場合、セクション自体を非表示にする。

### 3.2 これからの家事セクション

今日以降に予定されている自分担当のタスクを日付ごとにグループ表示する。

| 要素 | 内容 |
|---|---|
| セクションヘッダー | 「これからの家事」ラベル + 説明文 + 未対応件数 |
| フィルタ | すべて / 今日 / 1週間 の3種類（フロント側で絞り込み） |
| 日付グループ | 日付（曜日）+ 件数 |
| タスクカード | 家事名 + スワイプ操作（完了 / スキップ） |

### 3.3 タスクカードの操作

スワイプでタスクステータスを更新する。

| 操作 | 内容 |
|---|---|
| 右スワイプ（完了） | タスクのステータスを完了に更新。カードをリストから除去 |
| 左スワイプ（スキップ） | タスクのステータスをスキップに更新。カードをリストから除去 |

- スワイプ中はカードが追従してドラッグし、しきい値（カード幅の30%）を超えたらアクション確定
- 操作確定後にトースト通知を表示

## 4. API

既存APIをそのまま利用する。

| メソッド | エンドポイント | 用途 |
|---|---|---|
| GET | `/api/users/me/profile` | ログイン中のユーザーID取得。担当者フィルタに使用 |
| GET | `/api/housework-tasks` | 世帯のタスク全件取得（バックエンドは世帯全員分を返す）。フロント側で `assigneeUserId == currentUserId` フィルタを実施（未割当タスクは含める） |
| PATCH | `/api/housework-tasks/{taskId}/status` | タスクステータス更新（完了 / スキップ） |
| PATCH | `/api/housework-tasks/bulk-status` | 過去タスクの一括完了 |

**担当者フィルタの注意点**: バックエンドの `/api/housework-tasks` は世帯全員分のタスクを返す。`/api/users/me/profile` と並行して取得し、`assigneeUserId == currentUserId` で絞り込む。`assigneeUserId` が null（未割当）のタスクは自分のリストに含める。

## 7. モバイル固有の考慮点

### スワイプ操作

| 操作 | 背景色 | ラベル |
|---|---|---|
| 右スワイプ（完了） | emerald-500（緑） | 完了 |
| 左スワイプ（スキップ） | slate-400（グレー） | スキップ |

- スワイプ中はカードの背景に色とアイコンが表示され、操作内容を視覚的にフィードバック
- タスクカードの枠線色はタスク種別で変化（過去タスク: rose系、今日のタスク: emerald系）
