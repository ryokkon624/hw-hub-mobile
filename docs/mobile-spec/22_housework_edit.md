# #22 家事編集 モバイル仕様

## 1. 概要

既存の家事マスタを編集・削除する画面。Web版SP版と同等の機能をそのまま踏襲する。フォーム構成は #21 家事新規作成 と同一だが、テンプレート選択ボタンは表示しない。

## 3. 機能仕様

### 3.1 初期表示

- `GET /api/houseworks/{houseworkId}` で家事データを取得し、フォームに初期値をセット
- 取得失敗時は #20 家事設定一覧 へリダイレクト

### 3.2 フォームフィールドと検証ルール

#21 家事新規作成と同一のバリデーションルール（テンプレート選択ボタンは非表示）。

| フィールド | 必須 | ルール |
|---|---|---|
| 家事名 | ✓ | 最大100文字 |
| 説明 | — | 任意テキスト |
| カテゴリ | ✓ | CLEAN / KITCHEN / GARDEN / GARBAGE / PET |
| 周期タイプ | ✓ | WEEKLY / MONTHLY / NTH_WEEKDAY |
| 曜日（WEEKLY時） | ✓ | 1曜日以上選択 |
| 月の日付（MONTHLY時） | ✓ | 1〜31（31=月末） |
| 第n週（NTH_WEEKDAY時） | ✓ | 1〜5 |
| 曜日（NTH_WEEKDAY時） | ✓ | 0〜6 |
| デフォルト担当者 | — | 世帯メンバーから選択（未設定可） |
| 開始日 | ✓ | YYYY-MM-DD |
| 終了日 | ✓ | YYYY-MM-DD、開始日以降 |

### 3.3 保存

- バリデーション通過後に `PUT /api/houseworks/{houseworkId}`
- 成功後スナックバーで通知し、#20 家事設定一覧 へ戻る

---

## 4. API

| メソッド | エンドポイント | 用途 |
|---|---|---|
| GET | `/api/houseworks/{houseworkId}` | 家事マスタ1件取得 |
| GET | `/api/households/{householdId}/members` | デフォルト担当者選択肢用 |
| PUT | `/api/houseworks/{houseworkId}` | 家事マスタ更新 |
