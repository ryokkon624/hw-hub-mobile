# #3 認証メール待機 モバイル仕様

## 1. 概要

サインアップ後、メール認証リンクのクリックを待つ案内画面。

送信先メールアドレスを表示し、メールが届かない場合の再送ボタンを提供する。再送後は60秒のクールダウンを設ける。

Web版との差異はなし。

---

│                                      │

## 3. 機能仕様

### 3.1 メールアドレスの引き渡し

- サインアップ画面 (#2) からナビゲーション引数（またはクエリパラメータ）でメールアドレスを受け取る
- メールアドレスがない状態でこの画面に遷移した場合はサインアップ画面 (#2) へリダイレクト

### 3.2 認証メール再送

| 状態 | ボタン表示 |
|---|---|
| 通常 | 「認証メールを再送する」（活性） |
| 送信中 | スピナー表示（非活性） |
| 再送完了後60秒 | 「再送 (N秒後)」（非活性）。1秒ごとにカウントダウン |

- 再送成功時: スナックバーで「メールを再送しました」と通知
- 再送失敗時: スナックバーでエラー通知

### 3.3 認証完了の検知

モバイルではリンクをクリックするとディープリンク経由でアプリが起動し、メール認証画面 (#4) が処理を担う。この画面自体はポーリングや自動遷移は行わない。

---

## 4. APIエンドポイント

| メソッド | エンドポイント | 用途 |
|---|---|---|
| POST | `/api/auth/email-verification/resend` | 認証メール再送 |

### POST /api/auth/email-verification/resend

```json
// Request
{ "email": "user@example.com" }

// Response 204 No Content
```

```
[認証メール待機画面]
  │
  ├─ ディープリンクで認証完了 ──────→ メール認証 (#4) ※OS経由でアプリ起動
  └─ ← 戻る ────────────────────────→ サインアップ (#2)
```
