# #5 招待受け取り モバイル仕様

## 1. 概要

世帯メンバーからの招待リンクをタップすると、ディープリンク経由でアプリが起動し、招待内容の確認と参加/辞退を行う画面。

ログイン状態によって表示内容が変化する:
- **未ログイン**: 招待情報を表示し、ログイン/サインアップへ誘導
- **ログイン済み**: 招待情報を表示し、「参加する」「辞退する」ボタンを表示

## 3. 機能仕様

### 3.1 ディープリンクによるトークン受け取り

| 項目 | 内容 |
|---|---|
| URLスキーム | `hwhub://invite/<TOKEN>` |
| go_router パス | `/invite/:token` |
| パラメータ | `token`（パスパラメータ） |

メール内リンク例:
```
hwhub://invite/abc123xyz
```

### 3.2 go_router 設定

```dart
GoRoute(
  path: '/invite/:token',
  builder: (context, state) {
    final token = state.pathParameters['token'] ?? '';
    return InvitationPage(token: token);
  },
),
```

### 3.3 招待情報の取得

- 画面表示時に `GET /api/household-invitations/:token` を呼び出す
- ローディング中はスピナーを表示
- エラー時（トークン無効・期限切れ等）はエラーメッセージと再読み込みボタンを表示

### 3.4 未ログイン時の処理

- `redirectAfterLogin` に現在の画面パス（`/invite/:token`）を保存
- ログイン/サインアップ完了後、自動的にこの画面へ戻る

### 3.5 ログイン済み時の処理

| 操作 | 処理 |
|---|---|
| 「参加する」タップ | `POST /api/household-invitations/:token/accept` → 成功後ホームへ遷移 |
| 「辞退する」タップ | `POST /api/household-invitations/:token/decline` → 成功後ログイン画面へ遷移 |

### 3.6 メールアドレス不一致警告

ログイン中のメールアドレスと招待先メールが異なる場合、警告バナーを表示する。参加操作は引き続き可能（強制はしない）。

---

## 4. APIエンドポイント

| メソッド | エンドポイント | 用途 |
|---|---|---|
| GET | `/api/household-invitations/{token}` | 招待情報取得 |
| POST | `/api/household-invitations/{token}/accept` | 招待受諾 |
| POST | `/api/household-invitations/{token}/decline` | 招待辞退 |

## 5. 画面遷移

```
[招待受け取り画面] ※ディープリンクで自動起動
  │
  ├─ 未ログイン → ログインして参加 ──→ ログイン (#1) ※redirect保存
  ├─ 未ログイン → 新規登録して参加 ──→ サインアップ (#2) ※redirect保存
  ├─ 参加する ──────────────────────→ ホーム (#10)
  └─ 辞退する ──────────────────────→ ログイン (#1)
```
