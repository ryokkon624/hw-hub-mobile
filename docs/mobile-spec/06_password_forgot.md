# #6 パスワード忘れ モバイル仕様

## 1. 概要

パスワードを忘れたユーザーがリセット用メールの送信を要求する画面。

メールアドレスを入力して送信すると、リセットメール送信完了画面 (#7) へ遷移する。情報漏えい防止のため、存在しないメールアドレスでも常に成功扱いとなる（バックエンド仕様）。

Web版との差異はなし。

## 3. 機能仕様

### 3.1 フォームバリデーション

| フィールド | ルール | ボタン非活性条件 |
|---|---|---|
| メール | 必須 | 空白のみ |

送信時バリデーションはAPIに委ねる（空欄は送信ボタン非活性で防止）。

### 3.2 メールアドレスの初期値

ログイン画面 (#1) からの遷移時に `email` クエリパラメータが渡される場合は、フォームに初期値としてセットする。

### 3.3 送信処理

- 送信中はボタンをスピナー表示に切り替え、再タップ不可
- 成功（204）→ リセットメール送信完了画面 (#7) へ遷移
  - 送信したメールアドレスを次画面への引数として渡す
- 失敗時はフォーム下部にエラーメッセージを表示

---

## 4. APIエンドポイント

| メソッド | エンドポイント | 用途 |
|---|---|---|
| POST | `/api/auth/password-reset/request` | パスワードリセットメール要求 |

### POST /api/auth/password-reset/request

```json
// Request
{ "email": "user@example.com" }

// Response 204 No Content
// ※存在しないメールでも 204 を返す（情報漏えい防止）
```
