# HwHub モバイル版 画面一覧

## 凡例
- **ディープリンク**: メールリンクからアプリを直接開く画面
- **image_picker**: カメラ/フォトライブラリからの画像選択が必要な画面
- **モバイル考慮**: Web版からUIや操作を変更する箇所

---

## 1. 認証系画面

| # | 画面名 | パス(Web版) | 権限 | 主な機能 | モバイル考慮 |
|:---|:---|:---|:---|:---|:---|
| #1 | ログイン | `/login` | 認証不要 | ・メール/パスワードでログイン<br>・パスワード忘れ画面へ遷移<br>・サインアップ画面へ遷移<br>・Googleログイン | Googleログインは`google_sign_in`パッケージ経由（ネイティブSDK） |
| #2 | サインアップ | `/signup` | 認証不要 | ・新規アカウント登録（メール/表示名/パスワード/言語）<br>・ログイン画面へ遷移 | |
| #3 | 認証メール待機 | `/signup/verify-wait` | 認証不要 | ・メール認証待ちメッセージの表示 | |
| #4 | メール認証 | `/email-verify` | 認証不要 | ・メール認証処理（トークン検証） | ディープリンクで受け取り |
| #5 | 招待受け取り | `/invite/:token` | 認証不要 | ・招待された世帯情報の確認<br>・招待の受諾（ログイン/新規登録/既存）<br>・辞退 | ディープリンクで受け取り |
| #6 | パスワード忘れ | `/password/forgot` | 認証不要 | ・パスワードリセットメールの送信要求 | |
| #7 | リセットメール送信 | `/password/reset-sent` | 認証不要 | ・送信完了メッセージの表示 | |
| #8 | パスワード再設定 | `/password/reset` | 認証不要 | ・新しいパスワードの設定 | ディープリンクで受け取り |
| #9 | 認証結果 | `/auth/result` | 認証不要 | ・認証アクション（パスワードリセット等）の結果表示 | |

## 対象外画面（管理画面）

管理者・スタッフ向け機能はPC運用を前提とするため、モバイル版の対象外とする。

| 画面名 | 除外理由 |
|:---|:---|
| 管理トップ | 管理者のみ利用。モバイルで使うシーンなし |
| ユーザー管理 | 同上 |
| ロール管理 | 同上 |
| 問い合わせ管理（管理者側） | 同上 |
| 家事テンプレート管理 | 同上 |
